# 강의 요약

생성형 AI가 단순 질의응답 중심이었다면, 에이전트 AI는 목표를 달성하기 위해 스스로 판단하고 행동하는 자율형 시스템으로 발전하고 있다.

에이전트 AI는 단순히 답변을 생성하는 것이 아니라 정보를 수집하고, 추론하고, 필요한 도구를 활용해 실제 업무를 수행할 수 있다.

기업 환경에서는 단일 에이전트보다 여러 개의 에이전트가 협업하는 멀티 에이전트 구조가 일반적이며, 이를 통해 실제 조직과 유사한 업무 체계를 구현할 수 있다.

그러나 에이전트의 자율성과 권한이 커질수록 새로운 보안 위협도 증가한다. 따라서 에이전트 설계 단계부터 보안, 거버넌스, 권한 관리 체계를 함께 고려해야 한다.

# 핵심 개념

## 에이전트 AI란?

에이전트 AI는 목표를 달성하기 위해 스스로 계획을 수립하고 행동하는 AI 시스템이다.

기존 AI가 사용자의 질문에 답하는 수준이었다면, 에이전트 AI는 목표 달성을 위해 필요한 작업을 스스로 수행할 수 있다.

---

## 에이전트의 3가지 핵심 요소

### 1. 추론 (Reasoning)

* LLM을 활용해 상황을 이해하고 판단
* 역할(Role)을 정의해 업무 범위와 책임 설정 가능

### 2. 메모리 (Memory)

* 단기 기억(Short-Term Memory)
* 장기 기억(Long-Term Memory)

과거 경험과 정보를 기반으로 지속적인 업무 수행 가능

### 3. 도구 (Tool)

* 외부 시스템 및 API 활용
* 실제 행동을 수행하는 실행 수단

---

## 에이전트의 핵심 권한

### 자율성

주어진 목표를 스스로 수행하는 능력

### 정보 수집

필요한 데이터를 수집하는 능력

수집 범위와 시점을 명확히 정의하지 않으면 정보 유출 위험이 발생할 수 있다.

### 행동 권한

에이전트가 실제로 수행할 수 있는 행동 범위

발표자는 이 권한 설정이 가장 중요하다고 강조했다.

# 멀티 에이전트 아키텍처

기업 환경에서는 여러 개의 에이전트가 협업하는 구조가 사용된다.

예시

* 분석 에이전트
* 리서치 에이전트
* 검토 에이전트
* 승인 에이전트
* 관리자 에이전트

각 에이전트가 역할을 나누어 수행하며, 상위 오케스트레이션 에이전트가 전체 업무를 조정한다.

이는 실제 조직 구조와 유사한 형태이다.

---

# 에이전트 동작 프로세스

1. Trigger

   * 작업 시작

2. Collect

   * 정보 수집

3. Reason

   * 데이터 분석 및 추론

4. Process

   * 데이터 저장 및 가공

5. Deliver

   * 결과 전달

6. Review

   * 결과 검증 및 반복 수행

이 과정은 지속적으로 반복되며, 필요 시 상위 승인 체계를 거친다.

# 에이전트 보안 위협

## 왜 보안이 중요한가?

에이전트는 단순히 답변을 생성하는 것이 아니라 실제 행동을 수행한다.

따라서 잘못된 권한 부여나 악성 입력은 직접적인 보안 사고로 이어질 수 있다.

---

## 주요 보안 위협

### 공급망 공격 (Supply Chain Attack)

허깅 페이스(Hugging Face) 등 오픈소스 저장소에서 다운로드한 모델이나 도구에 악성 코드가 포함될 수 있다.

### 프롬프트 인젝션 (Prompt Injection)

외부 문서나 웹페이지에 숨겨진 명령어를 에이전트가 실행하도록 유도하는 공격

특히 간접 프롬프트 인젝션은 탐지와 방어가 어렵다.

### 권한 상승 (Privilege Escalation)

에이전트가 허용된 범위를 넘어 추가 권한을 획득하는 공격

### 메모리 오염 (Memory Poisoning)

에이전트가 사용하는 기억 데이터가 조작되는 공격

### 정보 유출 (Data Leakage)

에이전트가 잘못된 사용자나 시스템에 민감 정보를 전달하는 사고

# 주요 시사점

에이전트 AI의 핵심은 단순한 생성형 AI가 아니라 실제 업무를 수행하는 자율 시스템이라는 점이다.

에이전트가 더 많은 권한을 가질수록 생산성은 높아지지만, 동시에 보안 위험도 함께 증가한다.

따라서 기업은 에이전트 도입 시 기능 개발뿐 아니라 권한 관리, 보안 검증, 거버넌스 체계를 함께 설계해야 한다.
