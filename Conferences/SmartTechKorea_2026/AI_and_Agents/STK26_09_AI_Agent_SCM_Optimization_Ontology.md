# 강의 요약

공급망 관리(SCM)는 수요 예측, 생산 계획, 재고 관리, 공급 대응 등 수많은 의사결정이 동시에 발생하는 복잡한 영역이다.

본 강연에서는 이러한 SCM 업무를 AI 에이전트가 수행하는 미래 운영 모델을 소개하였다.

특히 온톨로지(Ontology)를 기반으로 조직과 데이터의 관계를 구조화하고, React 엔진을 통해 사람처럼 사고하며 원인 분석과 대응 방안을 제시하는 AI 에이전트 구조를 설명하였다.

발표자는 AI가 단순한 질의응답 도구를 넘어 실제 조직의 역할을 수행하는 협업 파트너로 발전하고 있으며, 이를 통해 기업은 실시간 의사결정 체계를 구축할 수 있다고 강조하였다.

---

# 핵심 개념

## 1. AI 에이전트 기반 SCM

SCM 환경에서는 여러 부서가 동시에 협업한다.

AI 에이전트 역시 각 조직의 역할을 담당하는 형태로 설계된다.

대표적인 에이전트는 다음과 같다.

* 수요(Demand) 에이전트
* 공급(Supply) 에이전트
* 재고(Inventory) 에이전트
* 생산(Production) 에이전트
* 오케스트레이션(Orchestration) 에이전트

각 에이전트는 담당자처럼 행동하며 서로 협상하고 견제하면서 최적의 공급망 의사결정을 수행한다.

발표자는 이를 "AI가 수행하는 SOP 회의"에 비유하였다.

---

## 2. 온톨로지(Ontology)의 중요성

AI가 업무를 수행하기 위해서는 조직과 데이터 구조를 이해해야 한다.

온톨로지는 객체, 역할, 이벤트, 시간 흐름 간 관계를 구조화하는 방법이다.

예를 들면

* 부서
* 담당자
* 제품
* 주문
* 생산
* 재고

등의 관계를 체계적으로 정의한다.

---

## 온톨로지 디자인 패턴(ODP) 6가지

### 1. Relation

객체 간 관계 정의

### 2. Event

주문 발생, 생산 시작, 재고 변동 등 이벤트 정의

### 3. Rolling

주간, 월간, 분기별 시간 흐름 관리

### 4. Role

조직과 역할 구조 정의

### 5. Data Structure

데이터 구조 설계

### 6. Explanatory Method

구조를 이해하기 쉽게 설명하는 방법

---

## 온톨로지 구축 효과

온톨로지가 구축되면

* 재고 에이징 최적화
* 납기 준수율 향상
* KPI 관계 분석
* 공급망 병목 파악

등이 가능해진다.

# React 엔진

## React 엔진이란?

React 엔진은 사람처럼 생각하는 AI 의사결정 구조이다.

SCM의 PDCA 사이클과 유사하게 동작한다.

* Plan
* Think
* Execute
* Check

과정을 반복하며 최적의 결과를 찾는다.

---

## 기존 시스템과 차이

기존 시스템

> "수요가 증가했습니다."

AI React 엔진

> "수요 증가의 원인은 특정 프로모션 때문이며, 추가 생산이 필요합니다."

즉 단순 보고가 아니라

* 원인 분석
* 데이터 조회
* 가설 검증
* 대응 방안 제안

까지 수행한다.

# AI 에이전트 협업 구조

수요가 급증했다고 가정하면

### 수요 에이전트

수요 증가 예측

### 생산 에이전트

생산 능력 검토

### 재고 에이전트

재고 소진 가능성 분석

### 공급 에이전트

원자재 확보 여부 검토

### 오케스트레이션 에이전트

전체 결과를 종합하여 최적 의사결정 제안

---

이 과정에서

* 라인 증설
* 외주 생산
* 생산 우선순위 변경
* 재고 재배치

등의 시나리오가 자동 검토된다.

# AI 도입 전략

발표자는 AI 도입 시 기술보다 먼저 비즈니스 문제를 정의해야 한다고 강조하였다.

AI를 도입하기 전에

* 어떤 KPI를 개선할 것인가
* 어떤 프로세스를 최적화할 것인가
* 어떤 의사결정을 자동화할 것인가

를 먼저 정의해야 한다.

온톨로지 구축 또한 기술 프로젝트가 아니라 업무 구조를 정리하는 작업으로 접근해야 한다.

---

# 주요 시사점

AI 에이전트는 단순한 챗봇이 아니다.

각 조직의 역할을 수행하는 디지털 팀원이 될 수 있다.

이를 위해서는 조직 구조와 업무 흐름을 온톨로지 형태로 정의해야 하며, React 엔진과 같은 추론 구조를 통해 원인 분석과 대응 방안 제시가 가능해진다.

앞으로 기업용 AI는 질의응답보다 협업과 의사결정 자동화 영역에서 더 큰 가치를 창출할 것으로 보인다.

---

# 한 줄 정리

SCM의 미래는 단일 AI가 아니라 수요·공급·재고·생산 에이전트가 온톨로지 기반으로 협업하며 의사결정을 수행하는 멀티 에이전트 시스템이다.
